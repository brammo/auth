# Internationalization

Plugin-facing strings in templates use the `brammo_auth` translation domain:

```php
__d('brammo_auth', 'Log in');
```

Locale files live under `resources/locales/<locale>/brammo_auth.po`.

## Extracting strings (host application)

From a CakePHP application that loads this plugin, extract messages into the plugin catalog:

```bash
bin/cake i18n extract \
  --paths vendor/brammo/auth/templates \
  --output resources/locales \
  --domain brammo_auth
```

Adjust `--paths` if the plugin is installed elsewhere.

## Compiling catalogs

```bash
bin/cake i18n init -l bg
msgfmt resources/locales/bg/brammo_auth.po -o resources/locales/bg/brammo_auth.mo
```

Flash messages configured in `Auth.Messages` use the host application domain via `__()` and are translated by the consuming app, not this plugin domain.
